## Reference Case Pattern

The reference Word case provided by the user uses this rhythm:

1. Centered title and short subtitle.
2. Scenario description with business background and goals.
3. Multi-step evidence chain using `用户说 / AI做了什么 / 结果`.
4. A comparison or targeted design table.
5. Team reuse value by role.
6. Final summary with achievements, efficiency comparison, promotion value, and reusable prompt.

Keep that rhythm, but replace project-specific sections with the target user's real project, organization, and evidence.

## Avoid

- Do not write only a file list or changelog.
- Do not copy private content from a reference case unless it is directly relevant and allowed.
- Do not exaggerate time saved.
- Do not present unverified checks as completed.
- Do not include credentials, tokens, private hostnames, real personal data, or production-only secrets.
